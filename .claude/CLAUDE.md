# vibe-kit — 프로젝트 안내 (클로드 코드가 매 세션 읽음)

> 이 파일은 '조언'이다. 클로드가 읽고 따르려 하지만, 꼭 지켜야 하는 강제는 `.claude/hooks/` 가 맡는다.
> 참고: Andrej Karpathy 의 CLAUDE.md 4원칙(가정 금지·최소 코드·수술적 수정·검증 반복)을 비개발자 프로젝트에 맞게 반영했다.

## 이 프로젝트는
바이브코딩 밀키트. 모노레포. 재료(통합)는 `packages/integrations/` 에 준비돼 있고, 사용자는 `plan.md` 대로 조립한다.

## 사용자에게 말할 때
- 사용자는 **비개발자**일 수 있다. 전문용어(API·DB·배포·커밋 등)는 그 자리에서 한 줄 비유로 풀어라.
- 코드를 보여주기 전, "이게 무슨 일을 하는지" 한국어 1~2줄 먼저.
- "간단합니다"로 뭉개지 말 것.

## 코딩 4원칙 (Karpathy — AI 는 못 짜서가 아니라 브레이크가 없어서 실수한다)
1. **가정하지 말고 물어라.** 요구가 애매하면 멈추고 되묻는다. 조용히 넘겨짚고 밀어붙이지 마라. 해석이 여럿이면 다 제시하고 사용자가 고르게 한다.
2. **최소 코드.** 시킨 문제를 푸는 가장 작은 코드만. 요청 안 한 기능·"나중을 위한" 유연성·안 일어날 상황의 예외 처리 금지.
3. **시킨 것만 고쳐라(수술적).** 인접 코드·주석·포매팅을 "개선"하지 마라. 안 망가진 걸 리팩토링하지 마라. 무관한 데드 코드는 발견해도 건드리지 말고 말만 해라.
4. **검증까지 반복.** "고쳤다"로 끝내지 말고, 실제로 되는지 확인(화면에서 눌러보기·테스트)할 때까지 스스로 돈다. 증거 없이 완료라 말하지 마라.

## 작업 방식
- **먼저 `plan.md` 를 읽어라.** 거기 블럭 순서대로 하나씩 만든다.
- 한 번에 한 블럭만. 끝나면 `plan.md` 체크박스를 갱신한다.
- 막히면 "더 작게 쪼개기". 큰 걸 통째로 시도하지 마라.

## 스택 규칙
- 웹 = `apps/web` (Next.js App Router) · 모바일 = `apps/mobile` (Expo).
- 저장이 필요하면 `packages/integrations/supabase` 를 쓴다. 아주 간단하면 `google-sheets`.
- 키는 항상 `.env` 에서 읽는다. 코드에 키를 박지 마라.
- 비교는 `===` / `!==` 만. `==` / `!=` 금지. null 은 명시적으로 확인.

## 보안 기본값 (비개발자 프로젝트라도 이건 자동으로 지켜라)
> 사용자가 "보안 해줘"라고 안 해도, 아래는 기본으로 적용한다. 어기게 될 때만 사용자에게 알리고 확인받는다.

**프론트(화면)**
- 사용자 입력은 항상 검증한다: 빈 값·너무 긴 값·형식 안 맞는 값을 막고, 안내 문구를 띄운다.
- 사용자가 쓴 글을 화면에 넣을 때 HTML 로 그대로 심지 마라(XSS). React 기본은 안전하니 `dangerouslySetInnerHTML` 을 쓰지 마라.

**백엔드/DB(저장·처리)**
- **남의 데이터가 보이면 안 된다.** Supabase 테이블은 만들 때 RLS(행 수준 보안)를 켜고, "본인 것만 읽기/쓰기" 정책을 기본으로 건다. 사용자 데이터에 **공개(anon) 삭제·수정 정책을 열지 마라.**
- 삭제는 되도록 soft delete(`deleted_at` 표시), 진짜 삭제는 사용자 확인 후.
- 중요한 처리(결제·권한·소유자 확인)는 프론트가 아니라 서버(서버 액션·API)에서 다시 확인한다.
- 에러 메시지·로그에 비밀번호·키·개인정보를 노출하지 마라.

**공통**
- 시크릿은 `.env` 에만. 절대 코드·깃허브·로그에 노출 금지(`.gitignore` 확인).
- 결제·인증처럼 사고가 크면, 먼저 테스트 키/테스트 모드로 만들고 실제 전환은 맨 마지막에.
- 확신 없으면 `security-reviewer` 서브에이전트에게 검토를 맡기고(배포 전 권장), 또는 사용자에게 "이 부분은 보안이 민감하다"고 알린다.

## 안전
- `.env` 를 읽거나 출력하지 마라.
- 파괴적 명령(rm -rf, DB 삭제·TRUNCATE·대량 UPDATE)은 사용자 확인 없이 실행 금지. 특히 실서비스 DB.
