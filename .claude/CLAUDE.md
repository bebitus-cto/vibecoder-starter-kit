# vibe-kit — 프로젝트 안내 (클로드 코드가 매 세션 읽음)

> 이 파일은 '조언'이다. 클로드가 읽고 따르려 하지만, 꼭 지켜야 하는 강제는 `.claude/hooks/` 가 맡는다.

## 이 프로젝트는
바이브코딩 밀키트. 모노레포. 재료(통합)는 `packages/integrations/` 에 준비돼 있고, 사용자는 `plan.md` 대로 조립한다.

## 사용자에게 말할 때
- 사용자는 **비개발자**일 수 있다. 전문용어(API·DB·배포·커밋 등)는 그 자리에서 한 줄 비유로 풀어라.
- 코드를 보여주기 전, "이게 무슨 일을 하는지" 한국어 1~2줄 먼저.
- "간단합니다"로 뭉개지 말 것.

## 작업 방식
- **먼저 `plan.md` 를 읽어라.** 거기 블럭 순서대로 하나씩 만든다.
- 한 번에 한 블럭만. 끝나면 `plan.md` 체크박스를 갱신한다.
- 막히면 "더 작게 쪼개기". 큰 걸 통째로 시도하지 마라.

## 스택 규칙
- 웹 = `apps/web` (Next.js App Router) · 모바일 = `apps/mobile` (Expo).
- 저장이 필요하면 `packages/integrations/supabase` 를 쓴다. 아주 간단하면 `google-sheets`.
- 키는 항상 `.env` 에서 읽는다. 코드에 키를 박지 마라.

## 안전
- `.env` 를 읽거나 출력하지 마라.
- 파괴적 명령(rm -rf, DB 삭제)은 사용자 확인 없이 실행 금지.
